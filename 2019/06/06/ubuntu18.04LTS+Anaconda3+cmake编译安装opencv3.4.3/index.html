ubuntu18.04LTS+Anaconda3+cmake编译安装opencv3.4.3
从opencv官网release下载source包->opencv3.4.3。
解压到/opt。那么opencv3.4.3的路径为：/opt/opencv3.4.3
cd到opencv3.4.3路径下。

目的是为了防止因读写权限而造成编译失败。

编译：
接下来就是关键的CMAKE参数设置:
python: anaconda3的python3.6.5
因此设置参数：

解释：
-D PYTHON_DEFAULT_EXECUTABLE=/anaconda3/bin/python3\ #指定python 可执行路径,这个参数可能没必要，加上也
没事
-D BUILD_opencv_python3=ON
-D BUILD_opencv_python2=OFF #指定python3安装
-D PYTHON3_EXCUTABLE=/anaconda3/bin/python3\ #指定python 可执行路径
-D PYTHON3_INCLUDE_DIR=/anaconda3/include/python3.6m\ #指定python_include的路径
-D PYTHON3_LIBRARY=/anaconda3/lib/libpython3.6m.a\ #指定python_lib的路径，这个一定要对上号。因为编译过程中会寻找合适的PythonLib,版本对不上，安装就会失败（我的python版本3.6.5,PythonLib版本也要3.6.5.这个信息在编译过程开始可以看到）
-D PYTHON_NUMPY_PATH=~/anaconda3/lib/python3.6/site-packages … #指定python_packages
执行以上命令，一定要仔细看编译过程，如果遇到 IPPICV下载失败的问题（这个很常见）：
在opencv3.4.3目录下， ctrl+h 显示隐藏文件：.cache。 会有一个文件名字是带md5编码的ippicv文件。看文件后面对应什么版本，比如我的是：7db2d34b9efa1cd22ff0572-ippicv_2017u3_lnx_intel64_general_20180518， 那么我需要的版本是：ippicv_2017u3_lnx_intel64_general_20180518。google: ippicv，从github上下载(https://github.com/opencv/opencv_3rdparty)即可。然后把下载的文件和带md5的同名文件放在一起。接下来，将不带md5重命名成带md5编码的文件名即可，意思是你下载好了。
接下来重新执行以上的编译命令就好了。
